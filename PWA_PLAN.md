# Progressive Web App (PWA) Implementation Plan
## Pair-Prep Live Goal Tracker

---

## Why PWA? (Your Situation)
✅ **No app store fees** — Deploy directly to Vercel, users link from browser  
✅ **Instant updates** — Push new features without app store review  
✅ **Cross-platform** — iOS + Android from one codebase  
✅ **Offline-first** — Goals still visible when offline, sync when back online  
✅ **Home screen access** — Installs like native app (Android ≈ 100%, iOS ≈ 85%)  
✅ **Push notifications** — Android full support, iOS 16.4+ (limited)  
✅ **Zero hosted backend** — Firebase handles everything  

---

## Modified Tech Stack (Optimized for PWA)

| Layer | Choice | Why | Cost |
|-------|--------|-----|------|
| **Frontend** | React 18 + Vite | Built-in PWA support, fast DX | Free |
| **PWA Framework** | Vite PWA Plugin + Workbox | Auto-generates manifest, service worker, offline caching | Free |
| **Styling** | Tailwind CSS | Mobile-first, responsive | Free |
| **Realtime DB** | Firebase Realtime DB | Free tier: 100 connections, instant sync | Free |
| **Auth** | Firebase Auth + Google Sign-In | Session persistence across install | Free |
| **Push Notifications** | Firebase Cloud Messaging (FCM) | Free on Firebase plan | Free |
| **Hosting** | Vercel | Auto-deploys PWA, HTTPS (required), 100GB/month free | Free |
| **Offline Storage** | IndexedDB (via Firebase lib) | Browser native, automatic sync | Free |

### 2️⃣ **Service Worker** (Auto-generated by Vite PWA Plugin)
Handles:
- ✅ Cache-first strategy for static assets
- ✅ Network-first for API calls (Firestore)
- ✅ Background sync for goal updates
- ✅ Offline page if needed

### 3️⃣ **Offline-First Data Strategy**
```
User creates goal → IndexedDB (instant)
              ↓
        Service Worker watches
              ↓
        When online → Firebase sync
              ↓
        Real-time listener updates both users
```

### 4️⃣ **Push Notifications (Android & iOS)**

**Android:** Full support via Firebase Cloud Messaging
**iOS:** Requires:
- PWA installed (can't send from web)
- iOS 16.4+
- User grants permission

## Configuration Details

### **Vite PWA Plugin Setup** (`vite.config.ts`)
```typescript
import { defineConfig } from 'vite'
import react from '@vitejs/plugin-react'
import { VitePWA } from 'vite-plugin-pwa'

export default defineConfig({
  plugins: [
    react(),
    VitePWA({
      registerType: 'autoUpdate',
      includeAssets: ['favicon.ico', 'robots.txt'],
      manifest: {
        name: 'Pair-Prep',
        short_name: 'Pair-Prep',
        description: 'Track daily prep goals with friends',
        theme_color: '#3b82f6',
        icons: [
          {
            src: '/icon-192.png',
            sizes: '192x192',
            type: 'image/png'
          },
          {
            src: '/icon-512.png',
            sizes: '512x512',
            type: 'image/png',
            purpose: 'any'
          }
        ]
      },
      workbox: {
        globPatterns: ['**/*.{js,css,html,ico,png,svg}']
      }
    })
  ]
})
```

### **Firebase Setup (Realtime DB Rules)**
```json
{
  "rules": {
    "users": {
      "$uid": {
        ".read": "auth.uid == $uid || exists(root.child('users').child(auth.uid).child('friendId').val()) && root.child('users').child(auth.uid).child('friendId').val() == $uid",
        ".write": "auth.uid == $uid"
      }
    },
    "goals": {
      "$uid": {
        ".read": "auth.uid == $uid || exists(root.child('users').child(auth.uid).child('friendId').val()) && root.child('users').child(auth.uid).child('friendId').val() == $uid",
        ".write": "auth.uid == $uid &&  newData.hasChildren(['date', 'text', 'done'])"
      }
    },
    "streaks": {
      "$uid": {
        ".read": "auth.uid == $uid || exists(root.child('users').child(auth.uid).child('friendId').val()) && root.child('users').child(auth.uid).child('friendId').val() == $uid",
        ".write": "auth.uid == $uid"
      }
    }
  }
}
```

## Hosting on Vercel (Deployment Checklist)

1. **Push code to GitHub**
   ```bash
   git push origin main
   ```

2. **Import project to Vercel**
   - Visit vercel.com
   - Click "New Project"
   - Select your GitHub repo
   - Done! Auto-deploys on every push

3. **Set Environment Variables in Vercel Dashboard**
   ```
   VITE_FIREBASE_API_KEY=xxx
   VITE_FIREBASE_PROJECT_ID=xxx
   VITE_FIREBASE_MESSAGING_SENDER_ID=xxx
   VITE_FCM_PUBLIC_KEY=xxx  (for push notifications)
   ```

4. **Verify PWA**
   - Open deployed URL on phone
   - Chrome DevTools → Application → Manifest (check status)
   - Lighthouse PWA audit: Score should be 90+

## Potential Challenges & Solutions

### **Challenge 1: iOS Push Notifications**
**Problem:** iOS PWAs can't receive push when closed (only notifications.permission works when installed)  
**Solution:** Send in-app notifications for non-urgent alerts; push for critical updates  
**Workaround:** Suggest users install as PWA for better behavior

### **Challenge 2: Safari Support (iOS)**
**Problem:** Safari has limited service worker support pre-iOS 16.4  
**Solution:** App still works, just limited offline. Majority of users on iOS 16+  
**Data:** 85%+ iOS users on iOS 16+

### **Challenge 3: Offline Sync Edge Cases**
**Problem:** Multiple devices editing same goal could conflict  
**Solution:** Use Firebase timestamps + "last write wins" strategy; show conflict indicator

### **Challenge 4: Icon Generation**
**Problem:** Need multiple icon sizes (192×192, 512×512, maskable versions)  
**Solution:** Use `pwa-asset-generator` CLI to auto-generate from 1 source image
